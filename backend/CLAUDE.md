### Current Controllers

**BaseController** — shared logic for all controllers:
```ruby
module Api
  module V1
    class BaseController < ApplicationController
      before_action :find_bank_account

      private
      def find_bank_account
        unless params[:account_number].present?
          render json: { error: 'Account number is required' }, status: :unprocessable_entity
          return
        end

        begin
          account_number = Integer(params[:account_number])
          @bank_account = BankAccount.find_by_account_number(account_number)

          unless @bank_account
            render json: { error: 'We could not find a bank account with that number, please try again.' }, status: :not_found
          end
        rescue ArgumentError
          render json: { error: 'Account number must be a valid integer' }, status: :unprocessable_entity
        end
      end

      def set_transaction_amount
        unless params[:amount].present?
          render json: { error: 'Amount is required' }, status: :unprocessable_entity
          return
        end

        begin
          _transaction_amount = Float(params[:amount])

          if _transaction_amount <= 0
            render json: { error: 'Amount must be greater than zero' }, status: :unprocessable_entity
            return
          end

          @transaction_amount = _transaction_amount
        rescue ArgumentError
          render json: { error: 'Amount must be a valid number' }, status: :unprocessable_entity
        end
      end
    end
  end
end
```

**BalanceController** — returns account balance:
```ruby
module Api
  module V1
    class BalanceController < BaseController
      def show
        render json: { balance: @bank_account.balance }, status: :ok
      end
    end
  end
end
```

**DepositController** — handles deposits:
```ruby
module Api
  module V1
    class DepositController < BaseController
      before_action :set_transaction_amount

      def create
        @bank_account.balance += @transaction_amount
        @bank_account.save
        render json: {
          deposit_amount: @transaction_amount,
          message: 'Your money was deposited successfully!'
        }, status: :ok
      end
    end
  end
end
```

**WithdrawController** — handles withdrawals:
```ruby
module Api
  module V1
    class WithdrawController < BaseController
      before_action :set_transaction_amount, :balance_is_enough?

      def create
        @bank_account.balance -= @transaction_amount
        if @bank_account.save
          render json: { withdraw_amount: @transaction_amount, message: 'Your money was withdrawn successfully!' }, status: :ok
        else
          render json: { error: 'Transaction failed' }, status: :unprocessable_entity
        end
      end

      private
      def balance_is_enough?
        if (@bank_account.balance - @transaction_amount) < 0
          render json: { error: 'Your balance is not high enough for your transaction amount' }, status: :unprocessable_entity
          return
        end
      end
    end
  end
end
```

## What I Already Understand
- `rails g model` → generates migration + model file
- `rails db:migrate` → actually creates the table in the database
- `rails db:rollback` → undoes the last migration
- `rails db:seed` → inserts dummy/test data from `db/seeds.rb`
- Migrations = version control for the database
- Model file = Ruby class to talk to the table without writing SQL
- Validations = catch errors before hitting the database, with clear messages
- `presence: true` = field must have a value
- `uniqueness: true` = no duplicate values allowed
- `add_index ..., unique: true` = enforces uniqueness at the database level
- Routes = map a URL + HTTP method to a controller action
- `namespace` in routes = groups routes under a URL prefix (e.g. `/api/v1/`)
- Controllers = classes that handle requests and return responses
- `before_action` = runs a method before any action in the controller
- `private` in controller = internal method, not callable as a route
- `params[:key]` = value sent in the request
- `render json:` = sends a JSON response
- `@instance_variable` in controller = accessible across methods in the same request — no `attr_reader` needed
- `present?` = not nil and not empty
- `find_by_column_name` = auto-generated Rails method to query by that column
- Seed data should be idempotent (safe to run multiple times)
- API error messages should be short, clear, and professional
- `.to_f` is lenient (returns 0.0 on bad input) — `Float()` is strict (raises `ArgumentError`)
- `Integer()` is strict like `Float()` — use it when the value must be a whole number
- `begin / rescue ErrorName` = handle a specific known error
- `rescue => e` + `e.class` = find out what error was raised (use for debugging only, not production)
- `render` does not stop execution — always add `return` after it inside conditionals
- HTTP status codes: `:ok` (200), `:unprocessable_entity` (422), `:not_found` (404)
- `unless x` = cleaner Ruby alternative to `if !x`
- Controller validation = good UX and clear error messages
- Model validation = protects database integrity — keep both layers
- `@bank_account.save` returns `true`/`false` — check the result when failure is possible
- `binding.pry` = pauses execution and opens an interactive console to inspect variables

## Important Decisions Made
- Using `integer` for `account_number` (not string)
- `balance` defaults to `0` — no presence validation needed
- Uniqueness enforced at DB level via `add_index`, not column option
- Model named `BankAccount`, table is `bank_accounts`
- Controllers namespaced under `Api::V1`
- Shared account lookup and amount validation logic lives in `BaseController`
- `set_transaction_amount` is in `BaseController` but only called via `before_action` in controllers that need it
- Both controller-level and model-level balance validation are kept — they protect different things
- `DepositController` does not check `save` result — deposit can't fail the balance validation
- `WithdrawController` checks `save` result — withdrawal can fail if balance goes below 0
